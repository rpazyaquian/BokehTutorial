Installation¶

This section gives more details about the installation process of Bokeh.

Dependencies¶

Because the Bokeh client library is mostly concerned with providing a nice Python interface for generating JSON objects which are then consumed by the BokehJS library running in the browser, there shouldn’t be a hard dependency on any of the standard NumPy/SciPy stack. It is entirely possible to use Bokeh with plain Python lists of values.

The Bokeh plot server does take advantage of NumPy, and may have a hard dependency on several compiled libraries.

Ideally, you should have the following libraries installed:

  • NumPy
  • Flask
  • Redis
  • Requests
  • six
  • gevent
  • gevent-websocket
  • Pandas

These are best obtained via the Anaconda Python Distribution, which was designed to include robust versions of popular libraries for the Python scientific and data analysis stacks.

Installing¶

There are a few different ways to install Bokeh:

using conda¶

If you are already an Anaconda user, you should be able to run the command:

$ conda install bokeh

This will install the most recent published Bokeh release from the Continuum Analytics Anaconda repository, along with all dependencies.

using pip¶

Bokeh may also be installed using pip:

$ pip install bokeh

which will attempt to download and build bokeh and its dependences from PyPI.

from source¶

You can also clone the Bokeh Github repository:

$ git clone https://github.com/ContinuumIO/bokeh
$ cd bokeh
$ python setup.py install

The setup.py script will automatically install the pre-built Javascript for the runtime library, so you do not need Coffeescript or any Javascript development infrastructure to use Bokeh.

Note for Windows Users¶

If you are using Windows but, for some reason, are not using Anaconda, then installing redis may be challenging. We recommend grabbing the binaries from here: https://github.com/dmajkic/redis/downloads.

We would still strongly recommend that Windows users consider installing a distribution like Anaconda, since it is the most painless way to obtain the Python scientific/analytic stack.
